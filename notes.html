CSS盒模型详解
2024-03-15

今天深入学习了CSS盒模型，理解了content-box和border-box的区别。

盒模型是CSS中的一个核心概念，它描述了元素内容、内边距、边框和外边距如何一起决定元素的总体大小。

默认情况下，浏览器使用标准盒模型（content-box），元素的width和height只包括内容区域。使用border-box更符合直觉，因为它包括内容、内边距和边框。

/* 设置所有元素使用border-box */ * { box-sizing: border-box; }

通过使用border-box，可以更容易地控制元素大小，尤其是在复杂布局中。

JavaScript作用域理解
2024-03-20

今天学习了JavaScript的作用域和闭包，这是前端开发中非常重要的概念。

JavaScript有三种作用域类型：

  • 全局作用域
  • 函数作用域
  • 块级作用域（ES6新增）
// 全局作用域 const globalVar = '我在全局可见'; function exampleFunction() { // 函数作用域 const functionVar = '我只在函数内可见'; if (true) { // 块级作用域 let blockVar = '我只在if块内可见'; var functionScopedVar = '我在整个函数内可见'; } console.log(functionScopedVar); // 可以访问 //console.log(blockVar); // 错误：blockVar未定义 }

理解闭包是JavaScript高级应用的关键，它允许函数访问并操作函数外部的变量。

Flexbox布局技巧
2024-03-25

Flexbox是我最喜欢的CSS布局方式之一，今天总结了一些实用技巧。

居中元素的最简单方法：

.container { display: flex; justify-content: center; align-items: center; height: 100vh; /* 视口高度 */ }

创建灵活的导航栏：

nav { display: flex; justify-content: space-between; } .nav-links { display: flex; gap: 20px; }

一个常见的问题是在不同设备上保持布局的响应性。使用flex-wrap: wrap可以让元素在空间不足时自动换行。

React Hooks使用心得
2024-04-02

今天深入学习了React Hooks，特别是useState和useEffect。

useState用于在函数组件中添加状态：

function Counter() { const [count, setCount] = useState(0); return ( <div> <p>You clicked {count} times</p> <button onClick={() => setCount(count + 1)}> Click me </button> </div> ); }

useEffect用于处理副作用，如数据获取、订阅或手动修改DOM：

useEffect(() => { // 这里的代码会在组件挂载和更新后执行 document.title = `You clicked ${count} times`; // 返回的函数会在组件卸载或下一次effect运行前执行清理 return () => { document.title = 'React App'; }; }, [count]); // 仅在count更改时重新运行

依赖数组是useEffect的关键，它决定了effect何时重新运行。空数组表示仅在挂载和卸载时运行。

Git工作流总结
2024-04-10

今天梳理了前端开发中常用的Git工作流程，特别是分支管理策略。

常用的Git命令：

# 创建并切换到新分支 git checkout -b feature/new-feature # 提交更改 git add . git commit -m "添加新功能" # 推送到远程 git push origin feature/new-feature # 合并分支 git checkout main git merge feature/new-feature

在团队协作中，我习惯使用Git Flow工作流，它包括以下主要分支：

  • main/master：稳定的生产代码
  • develop：开发分支
  • feature/*：新功能开发
  • release/*：发布准备
  • hotfix/*：紧急bug修复

使用Pull Request（PR）进行代码审查是保证代码质量的好方法。
